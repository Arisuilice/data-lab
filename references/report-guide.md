# report-guide

## 1. 报告目标

报告不是流程记录，而是帮助用户理解：

- 数据说明了什么；
- 证据有多强；
- 能做什么判断；
- 不能做什么判断；
- 下一步该怎么做。

## 2. 默认结构

```markdown
# Analysis Report

## Executive Summary

## Data Overview

## Method

## Results

## Interpretation

## Limitations

## Reproducibility
```

若是中文任务，使用中文标题。

## 3. 执行摘要

300-500 字或 3-5 个 bullet。必须包含：

- 直接回答用户问题；
- 最重要发现；
- 最佳模型或最强证据；
- 关键限制；
- 下一步建议。

不要在摘要中堆流程名。

## 4. 结果表达

每个核心发现建议使用：

```markdown
### Finding N: 标题
- Evidence: 指标、模型、图表或统计检验
- Meaning: 这说明什么
- Limitation: 这不能说明什么
- Action: 可采取什么下一步
```

不需要 Evidence ID 系统，除非是 Project 级审计任务。

## 5. 建模报告

必须包含：

- baseline 对比；
- 最佳模型验证/测试指标；
- train-test gap；
- 特征重要性或解释；
- 误差分析；
- 泄漏检查说明。

## 6. 统计报告

必须包含：

- 样本量；
- 方法；
- 效应量；
- 不确定性；
- 因果边界。

## 7. 限制写法

限制不是客套话，应具体影响解释：

- 样本历史性；
- 样本偏差；
- 缺失与测量误差；
- 目标截尾；
- 横截面不能因果；
- 模型外推风险；
- 关键变量缺失。

## 8. 复现信息

报告末尾列出：

- 输入文件；
- 主脚本；
- 输出表格；
- 输出图表；
- 关键依赖版本；
- 随机种子。
